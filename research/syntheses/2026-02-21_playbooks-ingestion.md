## Core pattern stack
1. **Detect repetition + pain first** (frequency and severity outperform novelty).
2. **Score opportunity explicitly** (repetition, pain, time saved, automation, reusability, output clarity, adoption).
3. **Promote by threshold**:
   - program candidate at score >= 20
   - product candidate at score >= 26
4. **Productize when workflow stabilizes** (predictable steps + reusable outputs + low customization).
5. **Prioritize systems over features** (intake -> process -> output -> review -> improvement).

## SaaS-strength indicators
- Replaces manual workflow
- Becomes system of record / visibility layer
- Enforces consistency and reduces risk
- Owns operational artifacts
- Expands across users, volume, workflows, and verticals

## Actionable extraction
- Use scoring + detection checks before packaging.
- Treat automation + visibility + artifact ownership as multiplier signals.
- Favor horizontal core with vertical overlays.
