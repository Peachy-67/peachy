# Decision Principles for Reliable Agents

To build dependable agents, optimize for accuracy and sound judgment before speed.

Core operating rules:
- Pause for user approval before high-impact or irreversible changes.
- Make progress in small, testable increments.
- Check each result before moving to the next step.
- Call out uncertainty and risks early.
- Keep logs so decisions and outcomes are easy to audit.

Knowledge quality rules:
- A single note is a hypothesis, not a final truth.
- Confidence should increase only when the same idea appears repeatedly.
- Direct user confirmation should carry high weight.
- Trusted external references can validate claims.

Performance standard:
- Dependable outputs
- Traceable actions
- Fast correction when wrong
- Clear communication
