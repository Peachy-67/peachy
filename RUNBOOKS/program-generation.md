# Program Generation Runbook

Goal: turn business problems into sellable operational programs.

## 1) Problem definition structure
- Problem statement (who, what, impact).
- Desired outcome (measurable target).
- Current-state baseline (how work happens now).
- Success criteria (quality, speed, cost, risk).

## 2) Constraints capture
- Time, budget, staffing.
- Compliance/legal constraints.
- Tooling/data constraints.
- Adoption/change-management constraints.
- Non-negotiables vs flexible constraints.

## 3) Workflow design steps
1. Map current workflow and bottlenecks.
2. Define future-state workflow (roles, handoffs, decisions).
3. Add control points (quality checks, approvals, exceptions).
4. Define metrics and reporting cadence.
5. Pilot with one representative use case.

## 4) SOP generation rules
- One SOP per repeatable unit of work.
- Use clear trigger → steps → checks → output format.
- Include owner, SLA, exception path, escalation path.
- Keep step language action-oriented and testable.

## 5) Deliverable packaging formats
- **Template pack**: forms/checklists/decision templates.
- **SOP bundle**: role-based SOP set + governance notes.
- **Tool spec**: required workflows, fields, automations, integrations.

## 6) Example output structure
- Program brief (problem, goals, constraints)
- Operating model (workflow map + role matrix)
- SOP bundle (core + exception SOPs)
- Templates/checklists
- Tool spec (if needed)
- Pilot plan + KPI dashboard
- Rollout plan (training + handoff)
